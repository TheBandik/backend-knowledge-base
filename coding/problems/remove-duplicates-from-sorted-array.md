# Remove Duplicates from Sorted Array

Источник: [LeetCode](https://leetcode.com/problems/remove-duplicates-from-sorted-array)

## Задача

Дан отсортированный по неубыванию массив целых чисел `nums`.

Необходимо удалить дубликаты **in-place** так, чтобы каждый уникальный элемент встречался только один раз, и вернуть количество уникальных элементов `k`.

Особенности:

* массив уже **отсортирован**
* порядок элементов **нельзя менять**
* первые `k` элементов должны содержать уникальные значения
* элементы после `k` **не имеют значения**

---

## Примеры

```text
Input:
nums = [1,1,2]

Output:
2, nums = [1,2,_]
```

```text
Input:
nums = [0,0,1,1,1,2,2,3,3,4]

Output:
5, nums = [0,1,2,3,4,_,_,_,_,_]
```

---

## Ограничения

* `1 <= nums.length <= 3 * 10^4`
* `-100 <= nums[i] <= 100`
* `nums` отсортирован в неубывающем порядке

---

## Идея решения

Так как массив уже отсортирован, все дубликаты идут подряд.

Это позволяет использовать подход с двумя указателями:

* один (`k`) – указывает на позицию последнего уникального элемента
* второй (`i`) – проходит по массиву

Если текущий элемент отличается от последнего уникального – значит это новый уникальный элемент.

---

## Алгоритм

1. Инициализировать указатель:

   * `k = 0` – индекс последнего уникального элемента

2. Пройти по массиву с `i = 1` до конца:

   * если `nums[i] != nums[k]`:

     * увеличить `k`
     * записать `nums[i]` в `nums[k]`

3. Вернуть `k + 1`

---

## Сложность

* Время: `O(n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public int removeDuplicates(int[] nums) {
    int k = 0;

    for (int i = 1; i < nums.length; i++) {
        if (nums[k] != nums[i]) {
            k++;
            nums[k] = nums[i];
        }
    }

    return k + 1;
}
```

---

## Подводные камни

* забывают, что массив **отсортирован** → усложняют решение
* начинают `k` с `1` и ломают индексы
* возвращают `k`, а не `k + 1`
* пытаются использовать дополнительные структуры (что запрещено условием)

---

## Как объяснить на собеседовании

Используется подход с двумя указателями. Поскольку массив отсортирован, дубликаты идут подряд. Мы поддерживаем индекс последнего уникального элемента и перезаписываем массив, когда встречаем новое значение.
