# Best Time to Buy and Sell Stock

Источник: [LeetCode](https://leetcode.com/problems/best-time-to-buy-and-sell-stock)

---

## Задача

Дан массив `prices`, где `prices[i]` – цена акции в день `i`.

Необходимо выбрать:

* один день для покупки
* один день в будущем для продажи

Чтобы **максимизировать прибыль**.

Если прибыль получить нельзя – вернуть `0`.

---

## Примеры

```text
Input:
prices = [7,1,5,3,6,4]

Output:
5

Explanation:
Покупка по 1, продажа по 6
```

```text
Input:
prices = [7,6,4,3,1]

Output:
0

Explanation:
Цена только падает, выгодной сделки нет
```

---

## Ограничения

* `1 <= prices.length <= 10^5`
* `0 <= prices[i] <= 10^4`

---

## Follow-up

Найти:

* решение за `O(n)` времени
* решение за `O(1)` памяти
* объяснение, почему greedy здесь корректен

---

## Идея решения

Самое важное:

> мы не ищем глобальный минимум и максимум

Задача на самом деле:

> найти максимальное значение `(prices[j] - prices[i])`, где `i < j`

Ключевая идея:

Для каждого дня считаем, какую прибыль получим, если продадим **сегодня**.

Для этого нужно знать:

* минимальную цену **в прошлом**

---

## Алгоритм

1. Инициализировать:

   * `minPrice = +∞`
   * `maxProfit = 0`

2. Пройти по массиву слева направо

3. Для каждого `price`:

   * если `price < minPrice` → обновить `minPrice`
   * иначе:

     * посчитать `profit = price - minPrice`
     * обновить `maxProfit`

4. Вернуть `maxProfit`

---

## Сложность

* Время: `O(n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE;
    int maxProfit = 0;

    for (int price : prices) {
        if (price < minPrice) {
            minPrice = price;
        } else {
            maxProfit = Math.max(maxProfit, price - minPrice);
        }
    }

    return maxProfit;
}
```

---

## Подводные камни

* пытаются найти **глобальный минимум и максимум** → ломается порядок
* забывают, что важно `i < j`
* делают два вложенных цикла → `O(n²)`
* обновляют максимум без учёта минимума слева

---

## Альтернативные решения

1. **Brute force**

   * перебор всех пар `(i, j)`
   * сложность `O(n²)`

2. **Через максимальную подпоследовательность (Kadane)**

   * перейти к массиву разностей
   * решать как max subarray

3. **Динамическое программирование**

   * хранить состояния "купил / продал"
   * избыточно для одной транзакции

---

## Как объяснить на собеседовании

Мы проходим массив один раз и на каждом шаге знаем минимальную цену в прошлом. Для текущего дня считаем прибыль, если продать сегодня. Таким образом, мы фактически проверяем все возможные сделки, но без вложенных циклов, что даёт `O(n)` по времени и `O(1)` по памяти.
