# Rotate Array

Источник: [LeetCode](https://leetcode.com/problems/rotate-array)

## Задача

Дан массив целых чисел `nums` и число `k`.

Необходимо **сдвинуть массив вправо на `k` шагов**.

Особенности:

* сдвиг циклический
* `k` может быть больше длины массива
* требуется решение **in-place** (без доп. памяти)

---

## Примеры

```text
Input:
nums = [1,2,3,4,5,6,7], k = 3

Output:
[5,6,7,1,2,3,4]
```

```text
Input:
nums = [-1,-100,3,99], k = 2

Output:
[3,99,-1,-100]
```

---

## Ограничения

* `1 <= nums.length <= 10^5`
* `-2^31 <= nums[i] <= 2^31 - 1`
* `0 <= k <= 10^5`

---

## Follow-up

Найти:

* несколько решений
* решение за `O(n)` времени
* решение за `O(1)` памяти

---

## Идея решения

Самое важное, что часто упускают:

> поворот массива – это просто перестановка сегментов

Ключевой трюк:

1. Развернуть весь массив
2. Развернуть первые `k` элементов
3. Развернуть оставшиеся `n - k`

Почему это работает:

* после полного разворота хвост становится в начале
* остаётся только восстановить порядок внутри двух частей

---

## Алгоритм

1. Вычислить:

   * `n = nums.length`
   * `k = k % n` (важно!)

2. Развернуть весь массив

3. Развернуть:

   * от `0` до `k - 1`
   * от `k` до `n - 1`

---

## Сложность

* Время: `O(n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k = k % n;

    reverse(nums, 0, n - 1);
    reverse(nums, 0, k - 1);
    reverse(nums, k, n - 1);
}

private void reverse(int[] nums, int left, int right) {
    while (left < right) {
        int temp = nums[left];
        nums[left] = nums[right];
        nums[right] = temp;
        left++;
        right--;
    }
}
```

---

## Подводные камни

* забывают `k = k % n` → вылет за границы
* пытаются делать сдвиг по одному → `O(n * k)` (плохо)
* используют дополнительный массив → не проходит follow-up
* путаются с границами `k - 1`

---

## Альтернативные решения

1. **Дополнительный массив**

   * копируем элементы на новые позиции
   * просто, но `O(n)` память

2. **Циклические замены (cycle replacement)**

   * более сложная логика
   * тоже `O(1)` память

3. **Повторный сдвиг**

   * простой, но неэффективный (`O(n * k)`)

---

## Как объяснить на собеседовании

Мы используем трюк с разворотом массива. Полный разворот меняет порядок элементов, после чего мы отдельно разворачиваем две части, чтобы восстановить их внутренний порядок. Это позволяет сделать циклический сдвиг за `O(n)` и без дополнительной памяти.
