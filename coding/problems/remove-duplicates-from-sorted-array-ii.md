# Remove Duplicates from Sorted Array II

Источник: [LeetCode](https://leetcode.com/problems/remove-duplicates-from-sorted-array-ii)

## Задача

Дан отсортированный по неубыванию массив целых чисел `nums`.

Необходимо удалить дубликаты **in-place** так, чтобы каждый уникальный элемент встречался **не более двух раз**, и вернуть количество элементов `k`.

Особенности:

* массив уже **отсортирован**
* порядок элементов **нельзя менять**
* первые `k` элементов должны содержать корректный результат
* элементы после `k` **не имеют значения**
* запрещено использовать дополнительную память (`O(1)`)

---

## Примеры

```text
Input:
nums = [1,1,1,2,2,3]

Output:
5, nums = [1,1,2,2,3,_]
```

```text
Input:
nums = [0,0,1,1,1,1,2,3,3]

Output:
7, nums = [0,0,1,1,2,3,3,_,_]
```

---

## Ограничения

* `1 <= nums.length <= 3 * 10^4`
* `-10^4 <= nums[i] <= 10^4`
* `nums` отсортирован в неубывающем порядке

---

## Идея решения

Это усложнённая версия классической задачи про удаление дубликатов.

Ключевая мысль:

> мы разрешаем оставить **до двух одинаковых элементов**

Так как массив отсортирован, одинаковые значения идут подряд, значит можно контролировать количество повторений через уже записанные элементы.

Вместо сравнения с последним элементом, сравниваем с элементом **на позиции `k - 2`**:

* если текущий элемент равен `nums[k - 2]` → это уже **третье вхождение**, пропускаем
* иначе → записываем в массив

---

## Алгоритм

1. Если длина массива ≤ 2 → вернуть длину (ничего удалять не нужно)

2. Инициализировать:

   * `k = 2` – первые два элемента всегда допустимы

3. Пройти по массиву с `i = 2` до конца:

   * если `nums[i] != nums[k - 2]`:

     * записать `nums[i]` в `nums[k]`
     * увеличить `k`

4. Вернуть `k`

---

## Сложность

* Время: `O(n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public int removeDuplicates(int[] nums) {
    int k = 2;

    for (int i = 2; i < nums.length; i++) {
        if (nums[i] != nums[k - 2]) {
            nums[k] = nums[i];
            k++;
        }
    }

    return k;
}
```

---

## Подводные камни

* пытаются считать количество повторений явно → это лишнее
* сравнивают с `nums[k - 1]` вместо `k - 2` → ломается логика
* забывают, что **первые два элемента всегда валидны**
* добавляют лишнюю память (set / list), что запрещено

---

## Как объяснить на собеседовании

Массив отсортирован, поэтому дубликаты идут подряд. Мы используем два указателя и контролируем, чтобы каждый элемент появлялся не более двух раз. Для этого сравниваем текущий элемент с элементом на позиции `k - 2`. Если они равны, значит это уже третье вхождение, и мы его пропускаем.
