# Merge Sorted Array

Источник: [LeetCode](https://leetcode.com/problems/merge-sorted-array)

## Задача

Даны два отсортированных массива `nums1` и `nums2` в неубывающем порядке, а также числа `m` и `n`, обозначающие количество валидных элементов.

Необходимо объединить массивы в один отсортированный массив, результат должен быть записан **внутрь `nums1`**.

Особенности:

* `nums1.length == m + n`
* первые `m` элементов – валидные
* последние `n` элементов – служебные (заполнены 0)
* `nums2.length == n`

---

## Примеры

```text
Input:
nums1 = [1,2,3,0,0,0], m = 3
nums2 = [2,5,6], n = 3

Output:
[1,2,2,3,5,6]
```

```text
Input:
nums1 = [1], m = 1
nums2 = [], n = 0

Output:
[1]
```

```text
Input:
nums1 = [0], m = 0
nums2 = [1], n = 1

Output:
[1]
```

---

## Ограничения

* `0 <= m, n <= 200`
* `1 <= m + n <= 200`
* `-10^9 <= nums1[i], nums2[j] <= 10^9`

---

## Идея решения

Если пытаться вставлять элементы с начала массива, придётся сдвигать элементы `nums1`, что приведёт к ухудшению сложности.

Правильный подход – заполнять массив **с конца**, где уже есть свободное место.

---

## Алгоритм

1. Инициализировать три указателя:

   * `i = m - 1` – последний валидный элемент `nums1`
   * `j = n - 1` – последний элемент `nums2`
   * `k = m + n - 1` – позиция для записи

2. Пока `i >= 0` и `j >= 0`:

   * если `nums1[i] > nums2[j]`, записать `nums1[i]` в `nums1[k]`
   * иначе записать `nums2[j]` в `nums1[k]`
   * сдвинуть соответствующий указатель

3. Если остались элементы в `nums2`, скопировать их в `nums1`

4. Если остались элементы в `nums1`, ничего делать не нужно

---

## Сложность

* Время: `O(m + n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    int i = m - 1;
    int j = n - 1;
    int k = m + n - 1;

    while (i >= 0 && j >= 0) {
        if (nums1[i] > nums2[j]) {
            nums1[k] = nums1[i];
            i--;
        } else {
            nums1[k] = nums2[j];
            j--;
        }
        k--;
    }

    while (j >= 0) {
        nums1[k] = nums2[j];
        j--;
        k--;
    }
}
```

---

## Подводные камни

* попытка решать с начала массива → приводит к перезаписи данных
* забывают обработать остаток `nums2`
* ошибки с индексами (`k = m + n - 1`)

---

## Как объяснить на собеседовании

Используется подход с тремя указателями. Заполнение идёт с конца массива, чтобы избежать перезаписи данных. На каждом шаге выбирается больший элемент из двух массивов и помещается в текущую позицию.

---

## Follow-up

Задача решается за `O(m + n)`, так как каждый элемент обрабатывается ровно один раз.
