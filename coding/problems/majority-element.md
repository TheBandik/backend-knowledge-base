# Majority Element

Источник: [LeetCode](https://leetcode.com/problems/majority-element)

## Задача

Дан массив целых чисел `nums` размером `n`.

Необходимо вернуть **мажоритарный элемент**.

Мажоритарный элемент – это элемент, который встречается **более чем ⌊n / 2⌋ раз**.

Гарантируется, что такой элемент **всегда существует**.

---

## Примеры

```text
Input:
nums = [3,2,3]

Output:
3
```

```text
Input:
nums = [2,2,1,1,1,2,2]

Output:
2
```

---

## Ограничения

* `1 <= nums.length <= 5 * 10^4`
* `-10^9 <= nums[i] <= 10^9`

---

## Follow-up

Решить задачу:

* за `O(n)` по времени
* за `O(1)` по памяти

---

## Идея решения

Это классическая задача, решаемая алгоритмом **Boyer–Moore Voting Algorithm**.

Ключевая мысль:

> если мажоритарный элемент встречается больше половины раз, он “перевесит” все остальные элементы

Мы храним:

* `majority` – текущего кандидата
* `count` – его “вес”

Логика:

* если `count == 0` → выбираем нового кандидата
* если текущий элемент равен кандидату → увеличиваем `count`
* иначе → уменьшаем `count`

В итоге останется именно мажоритарный элемент.

---

## Алгоритм

1. Инициализировать:

   * `majority = 0`
   * `count = 0`

2. Пройти по массиву:

   * если `count == 0` → `majority = num`
   * если `num == majority` → `count++`
   * иначе → `count--`

3. Вернуть `majority`

---

## Сложность

* Время: `O(n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public int majorityElement(int[] nums) {
    int majority = 0;
    int count = 0;

    for (int num : nums) {
        if (count == 0) {
            majority = num;
        }

        if (num == majority) {
            count++;
        } else {
            count--;
        }
    }

    return majority;
}
```

---

## Подводные камни

* путают условие и делают `if (num != majority)` – это ломает алгоритм
* пытаются использовать `HashMap` → лишняя память `O(n)`
* забывают, почему алгоритм вообще работает (на собесе спросят)
* думают, что нужно второй проход – **здесь не нужно**, потому что гарантирован ответ

---

## Как объяснить на собеседовании

Мы используем идею “голосования”. Каждый раз, когда встречается тот же элемент – усиливаем его, когда другой – ослабляем. Так как мажоритарный элемент встречается чаще половины массива, он не может быть “перебит” другими значениями и в итоге останется кандидатом.
