# Remove Element

Источник: [LeetCode](https://leetcode.com/problems/remove-element)

## Задача

Дан массив целых чисел `nums` и число `val`.

Необходимо удалить **все вхождения `val` из массива `nums` in-place** и вернуть количество оставшихся элементов `k`.

Особенности:

* порядок элементов **можно менять**
* первые `k` элементов массива должны содержать все значения, не равные `val`
* элементы после `k` **не имеют значения**

---

## Примеры

```text
Input:
nums = [3,2,2,3], val = 3

Output:
2, nums = [2,2,_,_]
```

```text
Input:
nums = [0,1,2,2,3,0,4,2], val = 2

Output:
5, nums = [0,1,4,0,3,_,_,_]
```

---

## Ограничения

* `0 <= nums.length <= 100`
* `0 <= nums[i] <= 50`
* `0 <= val <= 100`

---

## Идея решения

Ключевая мысль – **нам не важно сохранять порядок**, значит можно использовать более эффективный подход, чем сдвиг элементов.

Если текущий элемент равен `val`, мы можем:

→ заменить его на элемент с конца массива
→ уменьшить "рабочую" границу массива

Это позволяет избегать лишних сдвигов.

---

## Алгоритм

1. Инициализировать два указателя:

   * `i = 0` – текущий элемент
   * `j = nums.length - 1` – конец "рабочей" области

2. Пока `i <= j`:

   * если `nums[i] == val`:

     * заменить `nums[i]` на `nums[j]`
     * уменьшить `j`
   * иначе:

     * увеличить `i`

3. Вернуть `i` как количество валидных элементов

---

## Сложность

* Время: `O(n)`
* Память: `O(1)`

---

## Реализация (Java)

```java
public int removeElement(int[] nums, int val) {
    int i = 0;
    int j = nums.length - 1;

    while (i <= j) {
        if (nums[i] == val) {
            nums[i] = nums[j];
            j--;
        } else {
            i++;
        }
    }

    return i;
}
```

---

## Подводные камни

* забывают, что **порядок можно менять** → делают лишние сдвиги
* ошибка в условии цикла (`i <= j`)
* не понимают, почему `i` не увеличивается при замене
* пытаются очистить "хвост" массива – это не требуется

---

## Как объяснить на собеседовании

Используется подход с двумя указателями. Если находим `val`, заменяем его элементом с конца массива и уменьшаем рабочую область. Если элемент подходит, просто двигаемся дальше. Порядок не важен, поэтому это оптимальный способ избежать сдвигов.

---

## Альтернативное решение (с сохранением порядка)

Если вдруг требуют сохранить порядок:

```java
public int removeElement(int[] nums, int val) {
    int k = 0;

    for (int num : nums) {
        if (num != val) {
            nums[k++] = num;
        }
    }

    return k;
}
```
