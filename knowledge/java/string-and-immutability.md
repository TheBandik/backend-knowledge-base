# String и неизменяемость в Java

## Общая характеристика String

Класс `String` представляет неизменяемую последовательность символов.

```java
String s = "hello";
```

Особенности:

* объекты класса `String` являются **immutable**;
* широко используются в стандартной библиотеке и API;
* имеют специальную оптимизацию – **string pool**.

---

## Понятие неизменяемости (immutability)

### Определение

Объект называется **неизменяемым (immutable)**, если после создания его состояние не может быть изменено.

---

### Пример

```java
String s = "hello";
s.concat(" world");
```

Результат:

* исходный объект `"hello"` не изменяется;
* создаётся новый объект `"hello world"`.

---

### Иллюстрация

```text
s → "hello"

после concat:

s → "hello"
новый объект → "hello world"
```

---

## Внутреннее устройство String

Современная реализация (начиная с Java 9):

```java
private final byte[] value;
```

Ключевые аспекты:

* `final` ссылка на массив;
* содержимое не изменяется после создания;
* операции создают новые объекты.

---

### Почему это гарантирует неизменяемость

* поле `value` недоступно извне;
* отсутствуют методы, изменяющие массив;
* любые операции возвращают новый `String`.

---

## String pool (пул строк)

### Определение

**String pool** – область памяти, где хранятся уникальные строковые литералы.

---

### Пример

```java
String a = "test";
String b = "test";
```

Результат:

```java
a == b // true
```

Обе переменные указывают на один объект из пула.

---

### new String()

```java
String a = new String("test");
String b = new String("test");
```

Результат:

```java
a == b // false
```

Создаются разные объекты в heap.

---

### Метод intern()

```java
String s = new String("test").intern();
```

Возвращает ссылку на объект из string pool.

---

## Почему String неизменяемый

### Причины

1. **Безопасность**

   * строки используются в:

     * classloader;
     * файловых путях;
     * сетевых операциях.

2. **Кэширование**

   * hashCode может быть вычислен один раз.

3. **String pool**

   * позволяет переиспользовать объекты.

4. **Потокобезопасность**

   * immutable объекты безопасны без синхронизации.

---

## Операции со строками

### Конкатенация

```java
String s = "a" + "b";
```

На этапе компиляции:

```java
String s = "ab";
```

---

### Конкатенация в runtime

```java
String s = a + b;
```

Компилируется в:

```java
new StringBuilder().append(a).append(b).toString();
```

---

### Поведение

Каждая операция:

* создаёт новый объект;
* старый остаётся неизменным.

---

## Сравнение строк

### equals()

```java
a.equals(b)
```

Сравнивает содержимое.

---

### ==

```java
a == b
```

Сравнивает ссылки.

---

### Пример

```java
String a = "test";
String b = new String("test");

a == b       // false
a.equals(b)  // true
```

---

## hashCode() у String

`String` переопределяет `hashCode()`.

### Особенности:

* вычисляется на основе содержимого;
* кэшируется после первого вызова;
* благодаря неизменяемости всегда остаётся корректным.

---

## Проблема производительности

```java
String s = "";
for (int i = 0; i < 1000; i++) {
    s += i;
}
```

Создаётся множество объектов.

---

### Решение

```java
StringBuilder sb = new StringBuilder();
for (...) {
    sb.append(i);
}
String s = sb.toString();
```

| Класс         | Особенность                 |
| ------------- | --------------------------- |
| StringBuilder | быстрее, не потокобезопасен |
| StringBuffer  | потокобезопасен             |

## Иммутабельность в целом

### Признаки immutable класса:

1. класс объявлен `final` (желательно);
2. поля `private final`;
3. отсутствуют методы изменения состояния;
4. возвращаются копии изменяемых объектов.

---

### Пример

```java
final class User {
    private final String name;

    public User(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
```

---

## Типичные ошибки

1. Использование `==` для сравнения строк
2. Создание строк через `new` без необходимости
3. Конкатенация в циклах
4. Непонимание string pool
