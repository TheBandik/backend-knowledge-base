# equals и hashCode в Java

## Общая характеристика

Методы `equals()` и `hashCode()` определены в базовом классе `Object` и являются фундаментальными для:

* определения логического равенства объектов;
* корректной работы хеш-ориентированных структур данных.

По умолчанию:

* `equals()` реализует сравнение по ссылке (эквивалент `==`);
* `hashCode()` возвращает значение, связанное с идентичностью объекта.

---

## Понятие равенства объектов

### Физическое и логическое равенство

В языке Java необходимо различать два вида равенства:

1. **Физическое (reference equality)**
   Проверяется оператором `==` → сравниваются ссылки (один и тот же объект в памяти)

2. **Логическое (logical equality)**
   Определяется методом `equals()` → сравнивается содержательное состояние объектов

```java
User u1 = new User("Alice");
User u2 = new User("Alice");

u1 == u2       // false
u1.equals(u2)  // ожидается true
```

---

### Что именно сравнивается в equals()

Метод `equals()` не имеет универсальной логики. Он отвечает на вопрос:

> **Считать ли два объекта одинаковыми с точки зрения предметной области?**

Следовательно:

* разработчик самостоятельно определяет критерии равенства;
* сравниваются **значимые поля**, а не объект целиком.

---

### Пример

```java
class User {
    String name;
    int age;
}
```

Возможные стратегии:

* сравнение только по `name`;
* сравнение по `name` и `age`;
* сравнение по идентификатору (например, `id`).

Это решение зависит не от языка, а от бизнес-логики.

---

## Метод equals()

### Сигнатура

```java
public boolean equals(Object obj)
```

---

### Контракт equals()

Метод обязан удовлетворять:

1. **Рефлексивность**
2. **Симметричность**
3. **Транзитивность**
4. **Консистентность**
5. **Корректность при сравнении с null**

---

### Типичная реализация

```java
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null) return false;
    if (getClass() != o.getClass()) return false;

    User user = (User) o;
    return Objects.equals(name, user.name);
}
```

---

### Семантика шагов

* `this == o`
  – оптимизация: один и тот же объект

* проверка на `null`
  – предотвращение ошибок

* проверка типа
  – защита от некорректного сравнения

* сравнение полей
  – определение логического равенства

---

### Механизм Objects.equals()

```java
Objects.equals(a, b)
```

эквивалентно:

```java
(a == null) ? b == null : a.equals(b)
```

Обеспечивает безопасную работу с `null` и делегирует сравнение вложенным объектам.

---

## Метод hashCode()

### Определение

`hashCode()` – это функция, отображающая состояние объекта в целое число.

> Хеш-код – это **числовое представление объекта, вычисляемое на основе его значимых полей**.

---

### Важное уточнение

Хеш-код:

* не является уникальным идентификатором;
* не гарантирует различие объектов;
* используется как средство **ускорения поиска**.

---

### Контракт hashCode()

1. Равные объекты (по equals) обязаны иметь одинаковый hashCode
2. Неравные объекты могут иметь одинаковый hashCode
3. При неизменном состоянии объекта значение должно быть стабильным

---

### Реализация

```java
@Override
public int hashCode() {
    return Objects.hash(name);
}
```

---

### Ручная реализация

```java
@Override
public int hashCode() {
    int result = 17;
    result = 31 * result + (name == null ? 0 : name.hashCode());
    return result;
}
```

---

### Интерпретация

Хеш-код следует понимать как:

> **приближённый индекс, позволяющий быстро сузить область поиска объекта**

---

## Совместная работа equals() и hashCode()

### Разделение ролей

| Метод      | Назначение                   |
| ---------- | ---------------------------- |
| hashCode() | быстрый выбор области поиска |
| equals()   | точная проверка равенства    |

---

### Принцип работы в хеш-структурах

При добавлении объекта:

1. вычисляется `hashCode()`;
2. определяется индекс (корзина);
3. объект помещается в соответствующую структуру.

При поиске:

1. вычисляется `hashCode()`;
2. выбирается корзина;
3. вызывается `equals()` для проверки.

---

### Следствие

Если нарушен контракт:

* объекты могут не находиться;
* `equals()` может не вызываться вовсе.

---

## Коллизии

### Определение

Коллизия – ситуация, при которой разные объекты имеют одинаковый hashCode.

```text
"Alice" → 123
"Bob"   → 123
```

---

### Обработка

* объекты помещаются в одну корзину;
* поиск осуществляется через последовательное сравнение `equals()`.

---

### Вывод

* hashCode не обязан быть уникальным;
* корректность обеспечивается за счёт equals.

---

## Влияние изменяемости

Если объект используется как ключ:

```java
map.put(user, value);
```

и его состояние изменяется:

```java
user.setName("Bob");
```

то:

* изменяется hashCode;
* объект остаётся в прежней корзине.

Результат:

```java
map.get(user) // не находит объект
```

---

### Вывод

Объекты, используемые как ключи, должны быть:

* неизменяемыми;
* либо логически неизменяемыми после вставки.

---

## Сравнение == и equals()

| Оператор   | Назначение              |
| ---------- | ----------------------- |
| `==`       | сравнение ссылок        |
| `equals()` | сравнение по содержанию |

---

## Типичные ошибки

1. Переопределён только `equals()`
2. Использование изменяемых полей
3. Неверная проверка типов
4. Сравнение объектов через `==`
5. Несогласованность логики equals и hashCode
