# ООП

## Общая характеристика

Объектно-ориентированное программирование (ООП) – парадигма, в рамках которой программа представляется как совокупность взаимодействующих объектов, каждый из которых инкапсулирует:

* **состояние** (данные, поля);
* **поведение** (методы).

В языке Java ООП реализовано на уровне языка и среды выполнения (JVM) и является основным способом структурирования приложений.

---

## Объектная модель Java

### Объекты и ссылки

В Java:

* все сущности, кроме примитивных типов, являются объектами;
* переменные объектов содержат **ссылки (reference)** на объекты в heap-памяти.

```java
User u1 = new User("Alice");
User u2 = u1;
```

В данном примере:

* создаётся один объект;
* `u1` и `u2` ссылаются на один и тот же экземпляр.

### Передача параметров

Java использует **передачу по значению**:

* для примитивов – копируется значение;
* для объектов – копируется ссылка.

Это означает, что:

* сам объект не копируется;
* изменение состояния объекта внутри метода возможно, но переназначение ссылки – нет.

## Основные принципы ООП

### Инкапсуляция

Инкапсуляция – механизм сокрытия внутреннего состояния объекта и предоставления контролируемого доступа к нему через публичный интерфейс.

```java
class User {
    private int age;

    public void setAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException();
        }
        this.age = age;
    }
}
```

Ключевые аспекты:

* ограничение доступа (`private`, `protected`, `public`);
* контроль инвариантов;
* предотвращение некорректного использования объекта.

---

### Абстракция

Абстракция заключается в выделении существенных характеристик объекта и сокрытии деталей реализации.

```java
interface Storage {
    void save(String data);
}
```

Пользователь интерфейса:

* не знает, как именно реализовано хранение;
* работает только с контрактом.

---

### Наследование

Наследование позволяет создавать новые классы на основе существующих.

```java
class Animal {
    void move() {}
}

class Dog extends Animal {}
```

Особенности в Java:

* поддерживается **одиночное наследование классов**;
* допускается **множественная реализация интерфейсов**.

Назначение:

* повторное использование кода;
* построение иерархий типов.

Ограничение:

* повышает связность;
* может нарушать инкапсуляцию.

---

### Полиморфизм

Полиморфизм – способность объектов с одинаковым интерфейсом вести себя по-разному.

```java
Animal a = new Dog();
a.move();
```

В данном случае:

* статический тип – `Animal`;
* фактический тип – `Dog`;
* вызываемый метод определяется во время выполнения.

Это называется **динамическое связывание (dynamic binding)**.

---

## Связывание (Binding)

### Статическое связывание

Определяется на этапе компиляции.

Характерно для:

* перегрузки методов (overloading);
* `static` методов;
* `private` методов.

```java
void print(int x) {}
void print(String s) {}
```

---

### Динамическое связывание

Определяется во время выполнения.

Характерно для:

* переопределённых методов (overriding).

```java
class A {
    void f() {}
}

class B extends A {
    void f() {}
}
```

---

## Переопределение методов (Overriding)

Переопределение – изменение реализации метода родительского класса в подклассе.

```java
class A {
    void method() {}
}

class B extends A {
    @Override
    void method() {}
}
```

### Правила:

* сигнатура метода должна совпадать;
* уровень доступа нельзя уменьшать;
* допускается ковариантный возвращаемый тип.

```java
class A {
    Number get() { return 1; }
}

class B extends A {
    @Override
    Integer get() { return 1; }
}
```

### Не подлежат переопределению:

* `static` методы (они скрываются);
* `private` методы.

---

## Конструкторы и инициализация

### Вызов конструкторов

При создании объекта:

1. вызывается конструктор родительского класса;
2. затем конструктор текущего класса.

```java
class A {
    A() {
        System.out.println("A");
    }
}

class B extends A {
    B() {
        System.out.println("B");
    }
}
```

Результат:

```
A
B
```

### Явный вызов super

Если у родителя нет конструктора без параметров:

```java
class A {
    A(int x) {}
}
```

необходимо явно вызвать:

```java
class B extends A {
    B() {
        super(10);
    }
}
```

---

### Порядок инициализации

1. static поля;
2. static блоки;
3. instance поля;
4. instance блоки;
5. конструктор.

---

## Ключевые ключевые слова: this и super

### this

Ссылка на текущий объект.

Используется для:

* обращения к полям;
* вызова другого конструктора.

```java
User(String name) {
    this.name = name;
}
```

---

### super

Ссылка на родительский класс.

```java
super.method();
```

Используется для:

* вызова методов родителя;
* вызова конструктора родителя.

---

## Поля и методы: различие поведения

```java
class A {
    int x = 1;
}

class B extends A {
    int x = 2;
}

A obj = new B();
System.out.println(obj.x);
```

Результат:

```
1
```

### Объяснение:

* поля определяются на этапе компиляции;
* методы – во время выполнения.

Следовательно:

* **поля не полиморфны**;
* **методы полиморфны**.

---

## Базовый класс Object

Все классы в Java неявно наследуются от `Object`.

Основные методы:

* `toString()`
* `equals()`
* `hashCode()`
* `clone()`
* `finalize()` (устарел)

Метод `toString()` переопределяется для представления объекта в строковом виде:

```java
@Override
public String toString() {
    return "User{name='" + name + "'}";
}
```

---

## Модификаторы доступа

| Модификатор | Область видимости    |
| ----------- | -------------------- |
| private     | только внутри класса |
| default     | внутри пакета        |
| protected   | пакет + наследники   |
| public      | без ограничений      |

---

## Ключевое слово final

Используется для ограничения изменений:

* `final class` – запрещает наследование;
* `final method` – запрещает переопределение;
* `final variable` – значение не может быть изменено.

Важно:

* `final` ссылка не означает неизменяемость объекта.

---

## Композиция и наследование

### Наследование

Отношение «is-a».

Недостатки:

* высокая связность;
* зависимость от реализации базового класса.

---

### Композиция

Отношение «has-a».

```java
class Car {
    private Engine engine;
}
```

Преимущества:

* гибкость;
* возможность изменения поведения;
* снижение связности.

---

## Принцип подстановки Лисков (LSP)

Подкласс должен быть взаимозаменяем с базовым классом без изменения корректности программы.

Нарушение:

```java
class Rectangle {
    void setWidth(int w) {}
    void setHeight(int h) {}
}

class Square extends Rectangle {
    @Override
    void setWidth(int w) {
        super.setWidth(w);
        super.setHeight(w);
    }
}
```

Проблема:

* изменение поведения нарушает ожидания клиента.

---

## Типичные ошибки при использовании ООП

1. **Анемичная модель**

   * классы содержат только данные и геттеры/сеттеры;
   * логика вынесена наружу.

2. **Злоупотребление наследованием**

   * использование вместо композиции.

3. **Нарушение инкапсуляции**

   * избыточные геттеры/сеттеры.

4. **Смешение ответственностей**

   * один класс выполняет несколько ролей.
